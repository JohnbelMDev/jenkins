API tokens offer a way to make authenticated CLI or REST API calls. See our wiki for more details.
The username associated with each token is your Jenkins username.

Some good practices for keeping your API tokens secure are:
  • Use Unique Tokens per Application: Assign a unique token to each application. This allows for individual token revocation if an application is compromised, reducing potential security risks.
  • Regularly Regenerate Tokens: Regenerate tokens every 6 months or as per security requirements. Jenkins shows indicators for token age to prompt timely updates.
  • Protect Tokens as You Would Passwords: Tokens allow access with your Jenkins privileges. Safeguard them as securely as you would passwords to prevent unauthorized access.
  • Monitor Token Usage: Keep track of token usage. Jenkins provides the last use date and usage count for each token, assisting in identifying and revoking unused or compromised tokens.
  • Revoke Unused Tokens: Regularly review and revoke tokens that are no longer in use to reduce the attack surface by eliminating unnecessary active tokens.
  • Implement Strong Access Controls: Ensure only authorized users create and manage tokens. Use Jenkins' role-based access control for effective permission enforcement.
  • Keep Jenkins Updated: Regularly update Jenkins and plugins to the latest versions to ensure security patches for known vulnerabilities are applied.
Every time Jenkins is restarted, creation dates for unused legacy tokens reset, which may lead to inaccurate dates.
